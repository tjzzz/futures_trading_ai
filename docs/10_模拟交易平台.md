# 模拟交易平台

## 一、设计目标

构建一个完整的期货模拟交易系统，支持：
1. **虚拟账户管理** - 创建、管理多个模拟账户
2. **仓位管理** - 持仓品种、买卖操作、盈亏计算
3. **交易风控** - 保证金监控、止损止盈、风险控制
4. **策略执行** - 基于宏观分析信号的自动化交易
5. **绩效分析** - 收益曲线、胜率统计、风险指标

## 三、核心模块设计

### 1. 虚拟账户系统

**数据结构示例**：
```python
class VirtualAccount:
    account_id: str           # 账户ID
    balance: float            # 可用资金
    frozen_balance: float     # 冻结资金（保证金）
    positions: List[Position] # 持仓列表
    orders: List[Order]       # 订单列表
    risk_level: str          # 风险等级（保守/稳健/激进）
    created_at: datetime      # 创建时间
    updated_at: datetime      # 更新时间

class Position:
    symbol: str              # 合约代码（如AU2406）
    direction: str           # 方向（多/空）
    quantity: int            # 持仓数量
    avg_price: float         # 平均开仓价
    current_price: float     # 当前价格
    pnl: float               # 浮动盈亏
    margin: float            # 占用保证金
    open_time: datetime      # 开仓时间
```

### 2. 交易执行引擎

**主要功能**：
- 委托下单（支持限价、市价、条件单）
- 订单状态管理（待成交/部分成交/完全成交/已撤单）
- 成交回报处理
- 资金和持仓的实时更新

**接口设计**：
```python
class TradingEngine:
    def place_order(self, account_id: str, symbol: str, direction: str, 
                   order_type: str, price: float, quantity: int) -> str
    def cancel_order(self, order_id: str) -> bool
    def get_order_status(self, order_id: str) -> dict
    def get_account_info(self, account_id: str) -> dict
    def get_positions(self, account_id: str) -> List[dict]
```

### 3. 风险管理模块

**风控规则**：
1. **保证金监控**：实时计算保证金占用率，超过阈值预警
2. **止损止盈**：
   - 固定止损止盈
   - 移动止损（追踪最高/最低价）
   - 时间止损（持仓时间限制）
3. **仓位限制**：
   - 单品种最大持仓比例
   - 总持仓比例限制
   - 单日最大亏损限制

### 4. 策略执行模块

**信号处理流程**：
```
宏观信号（JSON） → 信号解析 → 方向过滤 → 技术指标计算 → 入场点选择 → 仓位计算 → 订单生成
```

## 四、模拟盘接入方案（整合自07_模拟盘研究调研）

### 1. 主流模拟交易平台对比

| 方案 | 接入方式 | 适用场景 | 优劣势 |
|------|----------|----------|--------|
| **SimNow (上期技术)** | CTP API 直连 | 机构/专业量化 | 最接近实盘，支持全套CTP协议，仿真环境稳定 |
| **快期模拟** | 快期客户端 + 接口 | 个人/半自动 | 界面友好，但接口封闭 |
| **文华模拟** | 文华API / 客户端 | 手动+自动化混合 | 模拟盘功能全，但API需付费授权 |
| **掘金量化** | SDK Python/C++ | 量化策略回测+模拟 | 自带回测+模拟环境，接口友好 |
| **天勤量化 (TQ)** | Python SDK | 量化交易 | 模拟盘免费，文档好，适合开发者 |
| **Backtrader + 模拟交割** | 自建模拟引擎 | 纯策略验证 | 不依赖外部，完全自主控制 |

### 2. 推荐方案：SimNow CTP 仿真 + 自建模拟引擎（双模式）

**架构设计**：
```
┌─────────────────┐
│  策略引擎模块     │ ← 技术分析信号生成
├─────────────────┤
│  CTP API 封装层  │ ← vnpy/ctp2 等 Python 封装
├─────────────────┤
│  网络层 (TCP)    │ ← SimNow 仿真环境
├─────────────────┤
│  中金所/大商所/   │
│  郑商所/上期所    │
│  (仿真撮合引擎)   │
└─────────────────┘
```

**Python 接入方案**：
- `vnpy` - 开源量化交易框架，自带CTP接口封装
- `xtquant` - 迅投QMT接口（有模拟盘）
- 自研轻量封装（基于 `ctp2` 或 `ctpapi`）

### 3. 与宏观分析师协作模式

```
┌────────────────────────────────────────────────┐
│               宏观分析师                         │
│  输出：宏观方向判断、大类资产观点、风险事件日历     │
│  格式：结构化的宏观信号（看多/看空/中性）+ 置信度  │
└──────────────┬─────────────────────────────────┘
               │ 信号输出 (JSON/结构化消息)
               ▼
┌────────────────────────────────────────────────┐
│            技术分析 & 操盘手（我）               │
│  输入：宏观大方向 + 数据中台提供的历史/实时数据    │
│  处理：结合技术指标择时、选入场/出场点位           │
│  输出：具体交易信号 + 仓位管理建议                 │
└──────┬───────────────────────┬─────────────────┘
       │                       │
       ▼                       ▼
┌──────────────┐     ┌──────────────────┐
│  模拟盘下单   │     │  策略评估/风控    │
│  (CTP仿真)   │     │  (绩效统计、归因)  │
└──────────────┘     └──────────────────┘
```

### 4. 与数据中台协作

**核心数据接口需求**：
| 数据类别 | 频率 | 用途 | 备注 |
|----------|------|------|------|
| 期货实时Tick | 逐笔 | 实时分析/交易 | 最细粒度 |
| 1min/5min K线 | 分钟级 | 技术指标计算 | 短线策略 |
| 15min/60min K线 | 15min/小时 | 中期趋势判断 | 波段策略 |
| 日K线 | 每日 | 大趋势、支撑阻力 | 长线参考 |
| 持仓/资金变动 | 事件驱动 | 风控、仓位管理 | 实时监控 |
| 历史K线数据 | 批量 | 策略回测 | 至少3年 |

**协作方式建议**：
- **方式一：消息队列（推荐）** - 数据中台推送实时行情到消息队列，技术分析引擎订阅消费
- **方式二：Feishu 文档/飞书表格（轻量）** - 适合初期快速验证
- **方式三：Pinecone/Redis（高频）** - 适合需要低延迟的场景

## 五、开发实施计划

### 第一阶段：基础框架（1-2周）
1. **虚拟账户系统** - 账户创建、资金管理、持仓记录
2. **订单管理** - 委托下单、撤单、状态查询
3. **基础风控** - 保证金计算、简单止损止盈
4. **Dashboard集成** - 持仓展示、盈亏计算

### 第二阶段：模拟盘接入（2-3周）
1. **SimNow环境搭建** - 注册账号、测试环境配置
2. **CTP API封装** - 使用vnpy或自研封装
3. **行情订阅** - 实时Tick数据接收
4. **委托下单** - 模拟盘交易接口对接

### 第三阶段：策略引擎（2-3周）
1. **宏观信号解析** - 解析宏观分析模块的输出
2. **技术指标计算** - MACD、RSI、布林带等
3. **信号融合** - 宏观+技术多因子策略
4. **自动化交易** - 基于信号的自动下单

### 第四阶段：高级功能（3-4周）
1. **高级风控** - 移动止损、仓位动态调整
2. **绩效分析** - 收益曲线、胜率统计、夏普比率
3. **回测框架** - 历史数据回测验证
4. **多账户管理** - 支持多个虚拟账户并行

## 六、API接口设计

### 1. 账户管理接口
- `POST /api/trading/accounts` - 创建虚拟账户
- `GET /api/trading/accounts/{account_id}` - 获取账户信息
- `PUT /api/trading/accounts/{account_id}/balance` - 调整资金
- `DELETE /api/trading/accounts/{account_id}` - 删除账户

### 2. 交易接口
- `POST /api/trading/orders` - 下单
- `DELETE /api/trading/orders/{order_id}` - 撤单
- `GET /api/trading/orders` - 查询订单
- `GET /api/trading/positions` - 查询持仓

### 3. 风控接口
- `GET /api/trading/risk/{account_id}` - 获取风险状态
- `POST /api/trading/risk/stop-loss` - 设置止损止盈
- `GET /api/trading/risk/alerts` - 获取预警信息

### 4. 策略接口
- `POST /api/trading/strategies` - 创建策略
- `PUT /api/trading/strategies/{strategy_id}` - 更新策略参数
- `POST /api/trading/strategies/{strategy_id}/execute` - 执行策略
- `GET /api/trading/strategies/{strategy_id}/performance` - 获取策略绩效
