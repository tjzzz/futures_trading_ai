# 事件驱动型宏观因子管理系统
> **作者：宏观分析师（AI助手）**

> 将新闻事件归因到四象限框架，追踪事件强度与时间轴
> 创建：2026.5.18 | v0.2（2026.5.19更新：新增自动化采集+时间轴管理）

---

## 一、核心设计理念

```
外部事件流 → 因子映射 → 强度评级 → 象限影响 → Dashboard可视化
```

不是"看新闻"，而是**把新闻转化成宏观因子的更新信号**。

---

## 二、事件 → 因子映射规则

每个新闻事件落地时，先归入四个象限之一，再标注强度影响。

### 🟢 货币锚（美元信用 + 利率）

| 事件类型 | 影响方向 | 示例 |
|---------|:--------:|------|
| 美债收益率突破关键位 | 美债信用↓→黄金↑ | 30Y破5% (2026.5.15) |
| 央行购金数据发布 | 购金↑→黄金↑ | 中国连续X月增持 |
| 美元指数大幅波动 | DXY↓→黄金↑ | 美联储偏鸽信号 |
| 美债拍卖遇冷/超额认购 | 需求↓→信用风险↑ | 拍卖bid-to-cover下降 |
| 美国信用评级调整 | 评级↓→黄金↑ | 惠誉/穆迪降级 |

### 🔵 宏观流动性（货币政策 + 流动性）

| 事件类型 | 影响方向 | 示例 |
|---------|:--------:|------|
| Fed利率决议 | 降息→流动性↑→金银↑ | 降息25bp |
| 美联储资产负债表变化 | 扩表→流动性↑→金银↑ | QE信号/缩表放缓 |
| FRA-OIS利差飙升 | 流动性紧张→金银短期↓长期↑ | 银行间市场压力 |
| 新美联储主席表态 | 偏鸽→金银↑ | 5月上任后首次讲话 |
| 美国财政政策 | 赤字↑→发债↑→长端↑ | 新财年预算案 |

### 🟠 风险偏好（地缘 + 市场情绪）

| 事件类型 | 影响方向 | 示例 |
|---------|:--------:|------|
| 地缘冲突升级/缓和 | 升级→避险↑→金银↑ | 美伊战争新动态 |
| 美股暴跌（单日-3%+） | 恐慌→先流动性危机→后避险 | 标普触发熔断 |
| VIX飙升 | 恐慌↑→金银短期承压长期受益 | VIX>30 |
| 谈判/停火/协议 | 缓和→避险↓→金银短期↓ | 停火协议签署 |
| 黑天鹅事件 | 极端不确定性→金银暴涨 | 类似俄乌开战初期 |

### 🔴 供需博弈（实物 + 挤兑）

| 事件类型 | 影响方向 | 示例 |
|---------|:--------:|------|
| COMEX交割压力 | 逼仓↑→银价↑ | 白银库存降至低位 |
| ETF持仓变化 | 流入↑→价格↑ | GLD/SLV出现大额申购 |
| 矿企生产中断 | 供给↓→价格支撑↑ | 主要银矿罢工 |
| 工业需求数据 | 需求↑→白银↑ | 光伏装机数据 |
| 印度/中国节日需求 | 实物需求↑→支撑↑ | 排灯节/春节备货 |

## 三、事件强度评级（S/A/B/C）

每件事落地时标注强度，决定触发什么级别的推送和Dashboard更新：

| 等级 | 定义 | 触发动作 | 示例 |
|:----:|------|---------|------|
| **S级** | 颠覆性事件，改变宏观叙事 | 实时推送 + Dashboard闪红 + 写专题分析 | 美债违约/美联储紧急降息/战争扩大 |
| **A级** | 重大事件，显著影响因子 | 实时推送 + Dashboard标记更新 | 30Y美债破5%/非农大幅不及预期 |
| **B级** | 重要事件，确认/强化趋势 | 次晨汇总 | Fed官员讲话/ETF数据变化 |
| **C级** | 常规事件，背景噪音 | 仅入库不推送 | 日常经济数据微幅波动 |

## 四、事件时间轴追踪

每个A级以上事件，建议按时间线持续追踪：

```
事件：30年美债收益率突破5%
开始：2026.5.15
状态：🔴 持续中 / ✅ 已结束 / ⏳ 观察中

时间轴：
├── 5.15 📌 30Y美债破5%，10Y破4.5%（触发点）
├── 5.16 📌 金银跟随下跌，市场恐慌加剧
├── 5.17 📌 无干预信号，收益率维持高位
└── ... （持续更新）

关联事件：
├── 美伊战争（2026.2.28起）
├── 新美联储主席上任（2026.5）
└── 美股开始回调
```

## 六、自动化采集与事件管理（v0.2新增）

### 6.1 采集流程

```
cron */30 * * * *
    ↓
news_collector.py 运行
    ├── 拉取4个RSS源（MarketWatch + CNBC Politics/Markets/Economy）
    ├── 合并去重（title+date）
    ├── 关键词匹配 → 自动归因到四象限
    ├── 关键词匹配 → 自动评级（S/A/B/C）
    ├── 写入 latest_feed.json（保留最近100条）
    └── 更新 dashboard_data.json 中的事件摘要

每30分钟自动触发（OpenClaw cron job）
S/A级新事件 → AI主动分析是否需推送镇哥
```

### 6.2 主动事件追踪库

手动维护的 `event_tracker.json` 包含：
- **活跃S级事件**：颠覆性叙事，实时追踪时间轴
- **活跃A级事件**：重大影响，每日更新进展
- **待观察事件**：B级中可能升级的候选
- **四象限事件分布摘要**：Dashboard联动用

### 6.3 时间轴更新策略

| 频次 | 动作 |
|:----:|------|
| 每30分钟 | 自动采集+自动归因+评级 |
| 每日（镇哥互动时） | 手动更新事件追踪库、补充解读、标记状态变更 |
| 事件触发时 | S/A级新事件推送解读 |

## 七、技术实现

### 7.1 采集脚本

```bash
# 文件位置：collectors/news_collector.py
# 功能：
#   1. 并发抓取4个RSS源
#   2. 关键词匹配归因到四象限（green/blue/orange/red）
#   3. 关键词匹配评级（S/A/B/C）
#   4. 写入 data/events/latest_feed.json
#   5. 更新 dashboard_data.json 事件摘要
```

### 7.2 事件追踪库

```bash
# 文件位置：data/events/event_tracker.json
# 结构：
#   active_events[] - 活跃的S/A级事件
#     → id, title, quadrant[], direction, level, status, started_at
#     → timeline[] - 时间轴（date, event, impact）
#     → linked_events[] - 关联事件ID
#     → summary - 宏观解读
#   events_to_watch[] - 待观察升级事件
#   quadrant_summary{} - 四象限事件摘要（Dashboard联动）
```

### 7.3 Dashboard事件集成

```json
// dashboard_data.json 新增 events 字段
{
  "events": {
    "active_s": ["30Y美债突破5%", "MFS破产信用危机"],
    "active_a": ["美伊战争僵局", "Fed内部降息分歧", "铝价看涨"],
    "quadrant_events": {
      "green": {"count": 3, "s_level": 2, "direction": "利多"},
      "blue": {"count": 1, "s_level": 0, "direction": "矛盾"},
      "orange": {"count": 5, "s_level": 1, "direction": "利多"},
      "red": {"count": 1, "s_level": 0, "direction": "偏多"}
    }
  }
}
```
