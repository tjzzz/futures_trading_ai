## 一、采集器架构总览

所有采集器统一继承 `BaseCollector`，遵循 **fetch → parse → write** 三阶段流水线：

```
fetch()                  parse()                   写入
  │                        │                         │
  │  从外部 API/CSV/HTML   │  解析为统一格式          │  write_snapshot()
  │  拉取原始数据          │  {snapshot_key,          │  → dashboard_data.json
  │                        │    snapshot_value,       │
  ▼                        │    history_row,          │  write_history_csv()
┌──────┐                   │    grain}                │  → data/history/xxx.csv
│ HTTP │                   │                          │
│ RSS  │──────────────────►│                          ▼
│ CSV  │                   │                    ┌──────────┐
└──────┘                   │                    │  持久化   │
                           │                    └──────────┘
```

## 二、BaseCollector 核心方法

| 方法 | 必须重写 | 返回值 | 说明 |
|------|---------|--------|------|
| `fetch()` | **是** | 原始数据（任意格式） | 从外部数据源拉取，含重试逻辑 |
| `parse(raw)` | **是** | dict（见下文格式） | 解析为结构化数据 |
| `run()` | 可选 | bool | 默认实现含完整流程，特殊场景可覆盖 |

### parse() 返回值格式

```python
{
    # ── 方案A：单字段快照（最常见）──
    "snapshot_key": "dxy",              # dashboard_data.json 中的字段名
    "snapshot_value": {                 # 字段值（可以是 dict 或 标量）
        "value": 99.148,
        "change": -0.179,
        "source": "yahoo_finance",
        "updated_at": "2026-05-21 00:45:39",
        "freshness": "实时",
    },

    # ── 方案B：多字段快照（需写入多个 dashboard 字段时）──
    "extra_snapshots": [                # 额外的字段，每项 (key, value)
        ("silver_price", {"value": 74.22, ...}),
        ("gold_silver_ratio", {"value": 60.56, ...}),
    ],

    # ── 历史行数据（写入 CSV）──
    "history_row": {                    # 键 = CSV 列名
        "dxy": 99.148,
        "us10y": 4.591,
        "vix": 17.87,
    },

    # ── 粒度（决定写入哪个 CSV 目录）──
    "grain": "minutely",                # "minutely" → data/history/minutely/
                                        # "daily"    → data/history/daily/
}
```

### write_snapshot vs 直接写快照

`BaseCollector.run()` 默认走 `write_snapshot()`（带文件锁），但**高频采集器（每5分钟）建议覆盖 `run()` 直接读写文件**，避免锁争用：

| 方式 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| `write_snapshot()` | 低频（日级/小时级） | 防并发脏写 | 有锁等待 |
| 直接读写文件 | 高频（分钟级） | 无锁争用，性能好 | 需自行处理并发 |

示例：[yahoo_finance.py:132-174](../collectors/yahoo_finance.py#L132-L174)

## 三、新增采集器步骤

### Step 1: 在 `collectors/` 下新建文件

```
collectors/
├── base_collector.py        ← 基类（无需修改）
├── gold_silver.py           ← 金银现货
├── yahoo_finance.py         ← DXY/US10Y/VIX 等高频
├── daily.py                 ← 日频指标（Treasury/FRED/VIX）
├── rss_news.py              ← RSS 新闻
├── aggregate_daily.py       ← 分钟→日聚合
├── backfill_2026.py         ← 历史回填
├── cleanup.py               ← 数据清理
└── your_new_collector.py    ← 新增文件
```

模板框架：

```python
#!/usr/bin/env python3
"""
{采集器名} — {简要描述}
- 源：{数据源 URL/名称}
- 粒度：{采集频率}
- 写入：{写入的目标字段/文件}
"""

import requests
from collectors.base_collector import BaseCollector


class YourNewCollector(BaseCollector):
    """{采集器说明}"""

    def __init__(self):
        # source_id 决定日志文件名和历史 CSV 文件名
        super().__init__("your_source_id")

    def fetch(self):
        """从数据源获取原始数据"""
        resp = requests.get(
            "https://api.example.com/data",
            timeout=15,
            headers={"User-Agent": "Mozilla/5.0"},
        )
        resp.raise_for_status()
        return resp.json()

    def parse(self, raw):
        """解析原始数据"""
        value = raw.get("price", 0)

        return {
            "snapshot_key": "your_field_name",
            "snapshot_value": {
                "value": round(value, 2),
                "source": "your_source",
                "updated_at": self._now(),
                "freshness": "实时",
            },
            "history_row": {
                "value": value,
            },
            "grain": "minutely",  # 或 "daily"
        }


if __name__ == "__main__":
    collector = YourNewCollector()
    collector.run()
```

### Step 2: 实现 `fetch()` — 数据源交互

注意事项：

- **超时**：始终设 `timeout`，建议 15s
- **User-Agent**：部分 API 需要模拟浏览器
- **重试**：高频采集器建议加重试（参考 `gold_silver.py:20-71`）
- **错误处理**：HTTP 错误 / JSON 解析失败 / 网络超时分别处理
- **返回格式**：返回 Python 对象（dict/list/str），不做格式转换

### Step 3: 实现 `parse()` — 数据解析

返回格式见 [第二节 parse() 返回值格式](#parse-返回值格式)。

字段命名规则：

| 场景 | 命名规则 | 示例 |
|------|---------|------|
| 快照字段 | 蛇形命名，全小写 | `treasury_10y`, `gold_futures` |
| 快照子字段 | 驼峰或蛇形 | `change_pct`, `prev_close` |
| CSV 列名 | 蛇形，与数据含义一致 | `gold_usd`, `silver_usd` |

freshness 标签规范：

| 标签 | 含义 | 适用 |
|------|------|------|
| `"实时"` | 延迟 < 30min | Yahoo Finance, gold-api |
| `"前一交易日"` | 延迟 1 天 | CBOE VIX 收盘 |
| `"延迟1天"` | 官方 T+1 | FRED SP500 |
| `"延迟2天"` | 官方 T+2 | FRED TIPS |
| `"延迟1~5天"` | 不定延迟 | FRED DXY |
| `"昨日数据"` | 隔日更新 | U.S. Treasury |

### Step 4: 注册数据源 → `data/sources/source_registry.json`

在 `source_registry.json` 的 `sources` 数组中添加新条目：

```json
{
  "id": "your-source-id",
  "name": "数据源中文名",
  "type": "api",
  "url": "https://api.example.com/endpoint",
  "frequency": "minutely",
  "collect_interval": 5,
  "collect_interval_unit": "minute",
  "delay": "real_time",
  "fields": ["field1", "field2"],
  "status": "active",
  "since": "2026-05-21"
}
```

### Step 5: 更新数据地图 `docs/02_数据中台.md`

3 处需要同步：

1. **一、数据源总览** — 新增行到数据源表格
2. **四、dashboard_data.json 字段说明** — 如果新增了快照字段
3. **五、采集器清单** — 新增行到采集器表格

### Step 6（可选）: 注册 URL 路由

如果新增了**前端页面**来展示数据，按 `docs/04_Dashboard开发.md` 流程操作。

## 四、历史数据 CSV 规范

### 目录规则

```
data/history/
├── minutely/          ← grain="minutely"
│   ├── gold_silver_minutely.csv
│   └── yahoo_finance_minutely.csv
└── daily/             ← grain="daily"
    ├── treasury.csv
    ├── tips.csv
    ├── dxy.csv
    ├── sp500.csv
    └── vix.csv
```

### CSV 格式规则

| 粒度 | 首列 | 后续列 | 保留策略 |
|------|------|--------|---------|
| `minutely` | `timestamp` (YYYY-MM-DD HH:MM:SS) | 指标值列 | 保留7天，由 `cleanup.py` 清理 |
| `daily` | `date` (YYYY-MM-DD) | 指标值列 | 永久保留 |

`BaseCollector.run()` 会自动根据 `grain` 决定目录和首列，无需手动处理。

然后在 `main()` 中注册。
