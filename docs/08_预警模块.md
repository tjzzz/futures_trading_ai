# 预警模块

## 一、设计目标

构建灵活、可定制的多级预警系统，支持：
1. **用户自定义预警规则** - 支持多指标、多条件组合
2. **多级阈值预警** - 黄色（关注）、橙色（提醒）、红色（警报）
3. **实时监控与触发** - 低延迟的事件检测和通知
4. **多渠道通知** - 飞书、邮件、Dashboard弹窗
5. **预警历史管理** - 事件记录、分析和复盘

## 三、核心模块设计

### 1. 预警规则系统

**规则定义格式**：
```json
{
  "rule_id": "price_breakout_alert",
  "name": "价格突破预警",
  "description": "当价格突破关键阻力位时触发",
  "symbols": ["XAUUSD", "XAGUSD"],
  "conditions": [
    {
      "type": "price",
      "operator": ">",
      "threshold": 2350.00,
      "duration": "1h",
      "window": "daily_high"
    },
    {
      "type": "volume",
      "operator": ">",
      "threshold": 1.5,
      "duration": "30m",
      "window": "avg_volume_20d"
    }
  ],
  "logic": "AND",
  "severity": "orange",
  "cooldown": "1h",
  "notifications": [
    {
      "channel": "feishu",
      "template": "price_breakout",
      "recipients": ["trading_team"]
    },
    {
      "channel": "email",
      "template": "detailed_alert",
      "recipients": ["risk_manager@company.com"]
    }
  ]
}
```

**规则类型支持**：
- **价格规则**：价格突破、大幅波动、异常价格
- **技术指标规则**：RSI超买超卖、MACD金叉死叉、布林带突破
- **宏观指标规则**：美债收益率变化、通胀数据异常、美元指数突破
- **复合规则**：多条件组合、自定义公式
- **时间规则**：特定时间触发、周期性检查

### 2. 多级阈值系统

**预警级别定义**：
| 级别 | 颜色 | 严重程度 | 触发条件 | 响应要求 |
|------|------|----------|----------|----------|
| **黄色预警** | 🟡 | 关注级 | 指标接近阈值 | 关注变化，无需立即行动 |
| **橙色预警** | 🟠 | 提醒级 | 指标突破阈值 | 准备应对，考虑调整仓位 |
| **红色预警** | 🔴 | 警报级 | 指标严重异常 | 立即行动，执行风控措施 |

**动态阈值调整**：
- **基于波动率** - 高波动时期自动放宽阈值
- **基于时间** - 不同交易时段使用不同阈值
- **基于市场状态** - 趋势市/震荡市使用不同参数
- **基于历史表现** - 根据误报率自动优化阈值

### 3. 实时监控引擎

**监控频率**：
- **高频监控**：价格数据（秒级/分钟级）
- **中频监控**：技术指标（分钟级/小时级）
- **低频监控**：宏观数据（小时级/日级）
- **事件监控**：新闻事件（实时推送）

**去重机制**：
- **时间窗口去重** - 同一规则在冷却期内不重复触发
- **相似事件合并** - 相近时间发生的同类事件合并通知
- **重要性过滤** - 低重要性事件批量报告

### 4. 通知分发系统

**通知渠道**：
- **飞书即时消息** - 快速推送，支持@提醒
- **邮件详细报告** - 包含详细数据和图表
- **Dashboard弹窗** - 网页端实时显示
- **短信通知** - 紧急情况备用渠道
- **API回调** - 集成到其他系统

**消息模板**：
```python
class AlertTemplate:
    def __init__(self):
        self.templates = {
            'price_breakout': {
                'feishu': "【价格突破预警】{symbol} 价格突破 {threshold}，当前 {price}",
                'email': self.generate_price_breakout_email,
                'dashboard': self.generate_price_breakout_card
            },
            'volatility_spike': {
                'feishu': "【波动率飙升】{symbol} 波动率异常，当前 {volatility}%",
                'email': self.generate_volatility_email,
                'dashboard': self.generate_volatility_chart
            }
        }
```

## 四、预警指标库

### 1. 价格类指标
| 指标名称 | 计算方式 | 预警场景 |
|----------|----------|----------|
| **价格突破** | 价格 > N日最高价 | 趋势启动信号 |
| **价格暴跌** | 单日跌幅 > X% | 风险事件发生 |
| **异常波动** | 波动率 > 历史N倍标准差 | 市场情绪极端 |
| **价格缺口** | 开盘价与昨日收盘价差距 > Y% | 隔夜重大事件 |

### 2. 技术指标类
| 指标名称 | 计算方式 | 预警场景 |
|----------|----------|----------|
| **RSI超买超卖** | RSI > 70 或 RSI < 30 | 反转信号 |
| **MACD背离** | 价格新高但MACD未新高 | 趋势衰竭 |
| **布林带突破** | 价格突破布林带上/下轨 | 极端波动 |
| **成交量异常** | 成交量 > 均量N倍 | 资金异动 |

### 3. 宏观指标类
| 指标名称 | 计算方式 | 预警场景 |
|----------|----------|----------|
| **美债收益率倒挂** | 10Y-2Y利差 < 0 | 经济衰退预警 |
| **通胀数据异常** | CPI同比变化 > 阈值 | 货币政策变化 |
| **美元指数突破** | DXY突破关键位 | 全球资产定价重估 |
| **VIX飙升** | VIX单日涨幅 > X% | 市场恐慌情绪 |

### 4. 复合指标类
| 指标名称 | 计算方式 | 预警场景 |
|----------|----------|----------|
| **风险情绪指数** | 多指标加权综合 | 整体市场风险 |
| **流动性压力指标** | 成交量+波动率+价差 | 市场流动性风险 |
| **相关性崩溃** | 资产间相关性突变 | 市场结构变化 |
| **板块轮动信号** | 不同品种相对强弱变化 | 资金流向变化 |

## 六、预警处理流程

### 1. 事件检测流程
```
数据更新 → 指标计算 → 规则匹配 → 条件评估 → 事件触发 → 去重检查 → 优先级计算 → 进入队列
```

### 2. 通知发送流程
```
事件队列 → 模板渲染 → 渠道路由 → 频率控制 → 发送执行 → 状态跟踪 → 送达确认 → 记录日志
```

### 3. 用户响应流程
```
收到通知 → 查看详情 → 评估影响 → 决策执行 → 反馈处理 → 规则优化
```

## 七、API接口设计

### 1. 规则管理接口
- `GET /api/alerts/rules` - 获取规则列表
- `POST /api/alerts/rules` - 创建新规则
- `PUT /api/alerts/rules/{rule_id}` - 更新规则
- `DELETE /api/alerts/rules/{rule_id}` - 删除规则
- `POST /api/alerts/rules/{rule_id}/test` - 测试规则
- `POST /api/alerts/rules/{rule_id}/backtest` - 回测规则

### 2. 预警事件接口
- `GET /api/alerts/events` - 获取预警事件列表
- `GET /api/alerts/events/{event_id}` - 获取事件详情
- `POST /api/alerts/events/{event_id}/ack` - 确认事件
- `POST /api/alerts/events/{event_id}/resolve` - 解决事件
- `GET /api/alerts/events/stats` - 获取事件统计

### 3. 通知配置接口
- `GET /api/alerts/notifications/channels` - 获取通知渠道
- `PUT /api/alerts/notifications/channels/{channel_id}` - 配置渠道
- `GET /api/alerts/notifications/templates` - 获取消息模板
- `POST /api/alerts/notifications/templates` - 创建模板
- `GET /api/alerts/notifications/history` - 获取通知历史

### 4. 监控状态接口
- `GET /api/alerts/status` - 获取监控状态
- `GET /api/alerts/status/rules` - 获取规则运行状态
- `GET /api/alerts/status/indicators` - 获取指标计算状态
- `GET /api/alerts/status/queues` - 获取队列状态
