# 方案：归因模块

> 日期：2026-05-21 | 状态：设计阶段

---

## 一、背景与问题

当前归因页面的问题：

- `query_indicator()` 只返回起止值、极值等基础统计，**没有真正的归因逻辑**
- HTML 中有一段硬编码的 mock 报告，点击查询后反而被覆盖成几个数字
- 宏观页面的 `analyze()` 有四象限规则分析（DXY/Treasury/VIX 阈值判断、信号综合），但那是**当前快照**的截面分析，不是**区间回顾**
- 短线和中长线归因的因子集一致，但不同时间粒度下因子权重不同——现有架构没有考虑这一点

---

## 二、归因本质

> 在指定时间窗口内，量化各因子对价格变动的贡献度，并给出主导因素的判断。

不是列举所有因子，而是回答：**"这次涨/跌主要是谁在驱动？"**

---

## 三、整体设计

### 3.1 架构

```
 attribution/                   ← 新增归因模块
 ├── __init__.py
 ├── engine.py                  ← 入口：统一归因接口
 ├── statistical.py             ← L1: 统计归因（相关性 + 贡献度）
 ├── event_matcher.py           ← L2: 事件匹配
 ├── report.py                  ← L3: 报告生成（规则模板 或 LLM）
 └── factors/                   ← 因子配置
     ├── gold.json              ← 黄金因子集
     └── silver.json            ← 白银因子集（可扩展）
```

### 3.2 三层递进

```
输入：品种(gold) + 时间窗口(2026-05-01~2026-05-20) + 粒度(日/分钟)
                 │
                 ▼
    ┌─────────────────────────────────┐
    │ L1: 统计归因                     │
    │   ├─ 各因子与金价的 pearson r     │
    │   ├─ 滚动 beta × Δ因子 = 贡献度   │
    │   └─ 按贡献度排名                 │
    └────────────┬────────────────────┘
                 │
    ┌────────────▼────────────────────┐
    │ L2: 事件匹配                     │
    │   ├─ 扫描窗口内 S/A 级事件        │
    │   ├─ 检测价格异动时刻              │
    │   └─ 匹配事件与异动               │
    └────────────┬────────────────────┘
                 │
    ┌────────────▼────────────────────┐
    │ L3: 综合报告                     │
    │   ├─ 合并统计归因 + 事件归因       │
    │   └─ 生成结构化归因报告            │
    └─────────────────────────────────┘
```

## 五、L1: 统计归因

### 5.1 算法

```
在指定窗口内，对每个因子 f：

  1. 取金价序列 P 和因子序列 F（对齐时间戳）
  2. 计算 pearson 相关系数 r = corr(P, F)
     → 衡量"同步程度"，|r| > 0.5 才算有效关联
  3. 计算因子变动幅度 ΔF = (F_end - F_start) / std(F)
     → 衡量"动了多少"
  4. 贡献度 = |r| × |ΔF| × sign(r × direction_sign)
     → 又同步、又动得多，才是主因
     → direction_sign: negative 因子取反（DXY 跌 = 黄金涨）
  5. 归一化：各因子贡献度 / max(|贡献度|) × 100%
```

### 5.2 输出示例

```json
{
  "target": "黄金 XAU/USD",
  "period": { "start": "2026-05-01", "end": "2026-05-20", "grain": "daily" },
  "price_change": { "from": 4420, "to": 4495, "absolute": 75, "pct": "+1.7%" },
  "driver_ranking": [
    { "factor": "dxy",   "contribution_pct": 68, "r": -0.78, "delta": -2.3,
      "detail": "DXY 从 101.5 跌至 99.2，贡献主要涨幅" },
    { "factor": "vix",   "contribution_pct": 22, "r": 0.45,  "delta": 4.0,
      "detail": "VIX 从 18 升至 22，避险情绪升温" },
    { "factor": "tips_10y", "contribution_pct": 10, "r": -0.65, "delta": 0.1,
      "detail": "实际利率变化不大，影响有限" }
  ]
}
```

### 5.3 接口

```python
def statistical_attribution(
    target: str,          # "gold" | "silver"
    start: str,           # "2026-05-01"
    end: str,             # "2026-05-20"
    grain: str = "daily"  # "daily" | "minutely"
) -> Dict:
    """执行统计归因"""
```

从配置加载 target 对应的因子集 → 按 grain 读取对应历史 CSV → 计算各因子的 r、Δ、贡献度 → 归一化排名。

## 六、L2: 事件匹配

### 6.1 算法

```
  1. 检测窗口内 gold 的异常波动日（|日涨跌幅| > 1.5σ）
  2. 对每个异常日，往前 12h 内搜索 event_tracker 中的 S/A 级事件
  3. 匹配到 → 标记为"疑似事件驱动"，附带事件标题和等级
```

### 6.2 输出示例

```json
{
  "matched_events": [
    { "date": "2026-05-15", "title": "30Y 美债收益率突破 5%",
      "level": "S", "quadrant": "货币锚",
      "price_impact": { "before": 4480, "after": 4420, "change": -60 },
      "direction": "短期承压" },
    { "date": "2026-05-18", "title": "美伊和谈破裂",
      "level": "A", "quadrant": "风险偏好",
      "price_impact": { "before": 4450, "after": 4495, "change": 45 },
      "direction": "避险推升" }
  ]
}
```

### 6.3 接口

```python
def match_events(
    target: str,
    start: str,
    end: str,
    price_series: List[Dict]  # L1 传过来的价格序列
) -> List[Dict]:
    """匹配窗口内的事件与价格异动"""
```

## 七、L3: 综合报告

### 7.1 规则模板模式（默认）

将 L1 + L2 的结构化数据填入模板：

```
{品种} {区间}，价格从 {start} 到 {end}，涨幅 {pct}。
主导因素：{因子1}(贡献 {n}%)，{因子2}(贡献 {n}%)。
事件影响：{S级事件数} 个 S 级事件。
结论：{一句话判断}
```

### 7.2 LLM 模式（可选）

将 L1 统计结果 + L2 事件匹配结果组装成 prompt，让 LLM 做叙事综合：

```
## 统计归因结果
金价变动: +1.7%
因子贡献排名:
  1. DXY: 贡献 +1.2% (r=-0.78, Δ=-2.3%)
  2. VIX: 贡献 +0.4% (r=0.45, Δ=+4pts)

## 事件匹配结果
1. 5/15 30Y美债破5%【S级】→ 当日金价波动 -1.3%
2. 5/18 美伊和谈破裂【A级】→ 当日金价波动 +1.0%

请据此生成一份结构化归因报告，分析主导因素。
```

LLM 不能修改 L1 的数据，只能基于统计结果做文字总结和事件解读。

## 八、与现有系统整合

### 8.1 替换 `query_indicator()`

```python
# analysis/engine.py
def query_indicator(self, indicator: str, start: str, end: str):
    if indicator in ("gold", "silver", "金银比", "XAU"):
        return AttributionEngine().run("gold", start, end)
    # fallback: 旧版 CSV 统计
    return self._rules_analyzer.query_indicator(indicator, start, end)
```

### 8.2 前端变更

`attribution.html` 中的 mock 数据移除，点击查询后展示 L1+L2+L3 的结构化结果：

```
┌─ 概览 ─────────────────────────────┐
│ 黄金 XAU/USD | 2026-05-01 ~ 05-20 │
│ 4420 → 4495  (+1.7%)              │
├─ 驱动排名 ─────────────────────────┤
│ 🥇 DXY      贡献 +1.2%  r=-0.78  │
│ 🥈 VIX      贡献 +0.4%  r=0.45   │
│ 🥉 TIPS     贡献 -0.1%  r=-0.65  │
├─ 事件匹配 ─────────────────────────┤
│ 🔴 5/15 30Y美债破5% (S级)         │
│ 🟡 5/18 美伊和谈破裂 (A级)         │
├─ 结论 ────────────────────────────┤
│ DXY 走弱是主因，VIX 避险辅助...     │
└────────────────────────────────────┘
```

### 8.3 与宏观页的关系

| 模块 | 职责 | 数据基础 |
|:----:|------|---------|
| 宏观分析 (`analyze()`) | **当前快照**的四象限截面判断 | dashboard_data.json |
| 归因 (`attribution()`) | **历史区间**的因子回顾 | 历史 CSV + 事件库 |

两者互补：宏观看现在是什么状态，归因看过去为什么这样走。
