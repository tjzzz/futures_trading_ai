# 预测模块

## 一、设计目标

构建多层次的价格预测系统，支持：
1. **短期预测（1-7天）** - 基于技术指标的趋势判断
2. **中期预测（1-4周）** - 基于宏观因子的方向预测
3. **长期预测（1-12个月）** - 基于基本面分析的趋势展望
4. **置信度评估** - 预测结果的可靠性量化
5. **回测验证** - 历史预测准确性评估

## 三、核心模块设计

### 1. 短期预测模块（1-7天）

**技术指标体系**：
- **趋势指标**：MA（5/10/20/60日）、EMA、MACD
- **动量指标**：RSI、Stochastic、CCI
- **波动率指标**：布林带、ATR
- **成交量指标**：OBV、成交量比率

**预测方法**：
1. **趋势延续性分析** - 当前趋势的强度和持续性判断
2. **关键位突破概率** - 支撑阻力位的突破可能性评估
3. **技术形态识别** - 经典价格形态的信号生成
4. **多时间框架验证** - 不同周期技术信号的协调性

**输出格式**：
```python
class ShortTermPrediction:
    symbol: str           # 品种代码
    time_horizon: str     # 时间范围（1d/3d/7d）
    direction: str        # 方向（看多/看空/震荡）
    target_price: float   # 目标价格
    stop_loss: float      # 止损价格
    confidence: float     # 置信度（0-1）
    reasoning: str        # 逻辑说明
    key_indicators: dict  # 关键指标值
```

### 2. 中期预测模块（1-4周）

**宏观因子模型**：
- **利率因子**：美债收益率曲线斜率（10Y-2Y）
- **通胀因子**：实际利率（名义利率-通胀预期）
- **汇率因子**：美元指数变化率
- **情绪因子**：VIX指数、黄金ETF持仓

**预测流程**：
```
宏观数据收集 → 因子标准化 → 相关性分析 → 模型训练 → 方向预测 → 置信度评估
```

**模型选择**：
- **线性回归** - 基础宏观因子模型
- **随机森林** - 非线性关系捕捉
- **梯度提升** - 多因子组合优化
- **集成学习** - 多模型结果融合

### 3. 长期预测模块（1-12个月）

**基本面分析框架**：
1. **供需分析**：
   - 全球产量/消费量平衡
   - 库存水平变化趋势
   - 产能扩张/收缩计划

2. **宏观周期定位**：
   - 经济周期阶段判断
   - 货币政策周期位置
   - 通胀周期分析

3. **结构性因素**：
   - 地缘政治风险评估
   - 技术变革影响
   - 监管政策变化

### 4. 预测融合与评估

**多模型融合策略**：
- **等权重融合** - 简单平均各模型结果
- **动态权重** - 基于历史表现调整权重
- **贝叶斯平均** - 考虑模型不确定性的加权

**置信度评估方法**：
1. **历史准确性** - 模型在过去相似场景下的表现
2. **信号强度** - 技术指标的背离程度
3. **市场一致性** - 不同时间框架信号的协调性
4. **数据质量** - 输入数据的完整性和时效性

```python
    def __init__(self):

            'real_rate': None,     # 实际利率
            'dxy_change': None,    # 美元指数变化
            'vix_level': None,     # VIX水平
        }
    
    def collect_factors(self) -> Dict[str, float]
    def train_model(self, X: np.array, y: np.array)
    def predict_direction(self, current_factors: Dict[str, float]) -> Tuple[str, float]
```

### 3. 预测融合引擎（示例结构）
```python
class PredictionFusionEngine:
    def __init__(self):
        self.models = {
            'technical': TechnicalModel(),
            'macro': MacroModel(),
            'fundamental': FundamentalModel(),
        }
        self.weights = self.calculate_weights()
    
    def fuse_predictions(self) -> PredictionResult:
        predictions = []
        for name, model in self.models.items():
            pred = model.predict()
            predictions.append((name, pred, self.weights[name]))
        
        # 加权融合
        fused = self.weighted_average(predictions)
        fused.confidence = self.calculate_confidence(predictions)
        return fused
```

## 七、回测框架

### 1. 回测指标
- **准确率** - 方向预测正确比例
- **盈亏比** - 平均盈利/平均亏损
- **夏普比率** - 风险调整后收益
- **最大回撤** - 历史最大亏损幅度
- **胜率** - 盈利交易比例

### 2. 回测流程
```
历史数据准备 → 模拟预测生成 → 交易信号转换 → 模拟交易执行 → 绩效统计 → 报告生成
```

### 3. 持续优化
- **参数调优** - 技术指标参数优化
- **模型更新** - 基于新数据重新训练
- **权重调整** - 根据近期表现调整模型权重

- `POST /api/prediction/backtest` - 执行回测
- `GET /api/prediction/backtest/results/{backtest_id}` - 获取回测结果
- `GET /api/prediction/performance` - 获取模型表现统计

### 3. 管理接口
- `PUT /api/prediction/models/{model_id}/weights` - 调整模型权重
- `POST /api/prediction/models/{model_id}/retrain` - 重新训练模型
- `GET /api/prediction/models` - 获取模型列表和状态

1. **基本面数据收集** - 供需平衡、库存数据

### 第四阶段：融合与优化（4周）
1. **多模型融合引擎** - 加权融合不同时间框架预测

4. **持续优化机制** - 模型参数自动调优

## 十、注意事项

1. **预测不是预言** - 所有预测都应附带置信度和风险提示
2. **模型需要持续优化** - 市场环境变化需要模型适应
3. **多时间框架验证** - 避免单一模型的过度依赖
4. **风险管理优先** - 预测用于指导而非替代风控
5. **透明可解释** - 预测逻辑应清晰可追溯

> 注：预测模块的设计强调多层次、多方法融合，通过技术、宏观、基本面三个维度的分析，提供从短期到长期的全面价格展望，为交易决策提供数据支持。
