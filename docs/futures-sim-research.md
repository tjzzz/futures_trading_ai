# 期货模拟盘接入 & 协作方案调研

## 一、国内期货模拟盘接入方案

### 1. 主流模拟交易平台

| 方案 | 接入方式 | 适用场景 | 优劣势 |
|------|----------|----------|--------|
| **SimNow (上期技术)** | CTP API 直连 | 机构/专业量化 | 最接近实盘，支持全套CTP协议，仿真环境稳定 |
| **快期模拟** | 快期客户端 + 接口 | 个人/半自动 | 界面友好，但接口封闭 |
| **文华模拟** | 文华API / 客户端 | 手动+自动化混合 | 模拟盘功能全，但API需付费授权 |
| **掘金量化** | SDK Python/C++ | 量化策略回测+模拟 | 自带回测+模拟环境，接口友好 |
| **天勤量化 (TQ)** | Python SDK | 量化交易 | 模拟盘免费，文档好，适合开发者 |
| **Backtrader + 模拟交割** | 自建模拟引擎 | 纯策略验证 | 不依赖外部，完全自主控制 |

### 2. 推荐方案：SimNow CTP 仿真

**SimNow** 是上海期货信息技术有限公司（上期技术）提供的仿真交易环境，与实盘CTP协议完全一致。

- **注册**: 官网注册 SimNow 账户
- **环境**: 7x24小时仿真环境（部分时段清算）
- **接入**: CTP API（C++/Python封装）
- **地址**: `180.168.146.187:10100`（7x24环境）
- **BrokerID**: 9999（仿真环境统一ID）

**Python 接入方案**：

```
┌─────────────────┐
│  策略引擎模块     │  ← 技术分析信号生成
├─────────────────┤
│  CTP API 封装层  │  ← vnpy/ctp2 等 Python 封装
├─────────────────┤
│  网络层 (TCP)    │  ← SimNow 仿真环境
├─────────────────┤
│  中金所/大商所/   │
│  郑商所/上期所    │
│  (仿真撮合引擎)   │
└─────────────────┘
```

**推荐Python库**：
- `vnpy` - 开源量化交易框架，自带CTP接口封装
- `xtquant` - 迅投QMT接口（有模拟盘）
- 自研轻量封装（基于 `ctp2` 或 `ctpapi`）

### 3. 备选方案：天勤量化

## 二、与宏观分析师协作模式

```
┌────────────────────────────────────────────────┐
│               宏观分析师                         │
│  输出：宏观方向判断、大类资产观点、风险事件日历     │
│  格式：结构化的宏观信号（看多/看空/中性）+ 置信度  │
└──────────────┬─────────────────────────────────┘
               │ 信号输出 (JSON/结构化消息)
               ▼
┌────────────────────────────────────────────────┐
│            技术分析 & 操盘手（我）               │
│  输入：宏观大方向 + 数据中台提供的历史/实时数据    │
│  处理：结合技术指标择时、选入场/出场点位           │
│  输出：具体交易信号 + 仓位管理建议                 │
└──────┬───────────────────────┬─────────────────┘
       │                       │
       ▼                       ▼
┌──────────────┐     ┌──────────────────┐
│  模拟盘下单   │     │  策略评估/风控    │
│  (CTP仿真)   │     │  (绩效统计、归因)  │
└──────────────┘     └──────────────────┘
```

### 宏观信号 → 技术策略 映射流程

1. **宏观分析师** 发出方向信号（例：上证指数中期看多，高置信度）
2. **技术分析工程师** 将宏观方向作为过滤器：
   - 只在该方向上找入场点（看多信号下只做多、不做空）
   - 结合技术指标（支撑/阻力、MACD、RSI等）精准择时
   - 给出止损止盈、仓位比例
3. **数据中台** 提供：
   - 实时行情推送
   - 多周期K线数据
   - 历史数据用于回测
   - 账户资金、持仓等状态

## 三、与数据中台协作

### 核心数据接口需求

| 数据类别 | 频率 | 用途 | 备注 |
|----------|------|------|------|
| 期货实时Tick | 逐笔 | 实时分析/交易 | 最细粒度 |
| 1min/5min K线 | 分钟级 | 技术指标计算 | 短线策略 |
| 15min/60min K线 | 15min/小时 | 中期趋势判断 | 波段策略 |
| 日K线 | 每日 | 大趋势、支撑阻力 | 长线参考 |
| 持仓/资金变动 | 事件驱动 | 风控、仓位管理 | 实时监控 |
| 历史K线数据 | 批量 | 策略回测 | 至少3年 |

### 协作方式建议

**方式一：消息队列（推荐）**
```
数据中台 → Kafka/RocketMQ → 技术分析引擎
                          ↕
                   宏观分析师推送
```

- 数据中台推送实时行情到消息队列
- 技术分析引擎订阅消费
- 宏观分析师信号走同样的消息通道
- 优势：解耦、异步、可扩展

**方式二：Feishu 文档/飞书表格（轻量）**
- 适合初期快速验证
- 数据中台写入飞书表格
- 技术分析读取分析
- 宏观信号以飞书文档结构输出

**方式三：Pinecone/Redis（高频）**
- 适合需要低延迟的场景
- 数据中台写入内存数据库
- 技术分析实时读取计算
