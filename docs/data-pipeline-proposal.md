# 数据采集管道设计方案
> **作者：宏观分析师（AI助手）**

## 一、目标

为金银宏观看板（Dashboard）建立自动化数据采集管道，覆盖框架四个象限的主要指标，实现"打开看板即看到最新数据"。

---

## 二、数据分层方案

```
┌─────────────────────────────────────────────┐
│           Dashboard 前端 (Flask)              │
│        每60秒轮询 dashboard_data.json          │
└──────────────────┬──────────────────────────┘
                   │ 读取
┌──────────────────▼──────────────────────────┐
│          dashboard_data.json                  │
│        统一数据文件，前端唯一数据源              │
└──────────────────┬──────────────────────────┘
                   │ 写入
┌──────────────────▼──────────────────────────┐
│           数据采集层 (Python)                  │
│  ┌───────────┐  ┌───────────┐  ┌──────────┐ │
│  │ 高频采集器│  │ 日频采集器│  │ 手工指标 │ │
│  │每5-30分钟 │  │ 每天1次   │  │ 手动输入 │ │
│  └───────────┘  └───────────┘  └──────────┘ │
└──────────────────┬──────────────────────────┘
                   │ HTTP GET
┌──────────────────▼──────────────────────────┐
│              数据源层                         │
│ gold-api.com │ Treasury CSV │ FRED │ CBOE   │
└─────────────────────────────────────────────┘
```

## 三、采集频率与指标分组

### 高频组（每5分钟 → 交易时段，非交易时段每30分钟）
- 伦敦金价 (XAU/USD) → gold-api.com
- 伦敦银价 (XAG/USD) → gold-api.com
- 金银比（计算得出）

### 日频组（每天北京时间 7:30，覆盖美东收盘后）
- 10Y / 30Y美债收益率 → Treasury CSV
- 2Y美债收益率 → Treasury CSV
- 实际利率 (10Y TIPS) → FRED
- 美元指数 DXY → FRED
- VIX 收盘价 → CBOE CSV
- 标普500 收盘价 → FRED

### 人工/事件驱动组（无需自动化）
- 央行购金Q1/Q2数据 → WGC季度报告
- FRA-OIS利差 → 金十/财联社关注
- COMEX白银库存 → 待调研替代源后补充
- Fed决议/新主席表态 → 新闻跟踪
- 地缘冲突状态 → 定性判断

## 四、数据流设计

```

FRED (DXY/TIPS/SP500) ──┤
                         │

┌─ data_collector.py ─────────────────────┐
│                                          │
│  def collect_high_freq():                │
│      # 每N分钟调用                        │
│      price_xau = fetch_gold_api('XAU')   │

│      ratio = price_xau / price_xag       │
│      write_json(...)                     │
│                                          │
│  def collect_daily():                    │
│      # 每天7:30调用                       │
```

## 五、错误处理策略

| 场景 | 处理方式 |
|------|----------|
| gold-api超时/挂掉 | 降级到Kitco页面抓取做兜底 |
| Treasury CSV当天未更新 | 使用前一日数据，标记"昨日数据" |
| FRED接口超时 | 跳过该字段保留旧值 |
| CBOE VIX CSV不可用 | 跳过保留旧值 |
| 所有外部源全挂 | Dashboard显示"待更新" |

## 六、技术栈

| 组件 | 选择 | 理由 |
|------|:----:|------|
| 采集语言 | Python 3 | 已有基础，requests库足够 |
| 定时触发 | Linux cron | 最简单可靠，无需额外依赖 |
| 数据存储 | JSON文件 | Dashboard前端直接读取 |
| 告警机制 | 日志文件 + 错误标记到JSON | 前端可通过标记知晓数据时效 |

## 七、实施计划

### Phase 1 — 核心管道（可独立运行）
1. 实现 `collect_high_freq()`：金银价格 + 金银比
2. 实现 `collect_daily()`：美债/VTX/VIX/SP500/DXY
3. 集成到 `data_collector.py`
4. 配置 cron 定时任务
5. 测试运行并观察1-2天

### Phase 2 — 增强
6. 加时间线标记（数据新鲜度标签）
7. 补充备用源降级逻辑
8. 增加历史趋势记录（每日快照存档）

### Phase 3 — 扩展
9. COMEX白银库存源解决后加入
10. 沪金/沪银溢价加入
11. 自定义告警阈值（10Y>4.5%标记警戒）

## 八、风险与局限

1. **FRED/Treasury数据延迟**：标普500、DXY等大概延迟1-2个工作日，日内波动无法覆盖。如需日内追踪，需要另找实时源（如CNBC页面抓取）
2. **COMEX白银库存**：CME官网国内网络不可达，目前没有找到稳定的免费替代源。如果镇哥知道哪个国内网站能看这个数据，可以补充
3. **金十/财联社**这些国内财经媒体有很好的实时数据，但页面结构复杂、反爬严格，投入产出比不高
4. **国外源稳定性**：gold-api目前免费且不限次数，但如果对方变更策略，需要备選方案
