## 一、功能概述

### 1.1 核心目标
- **主要功能**: 提供实时监控黄金/白银等关键指标在最近1小时/几分钟内的显著变化
- **核心价值**: 快速识别异常波动、提供归因分析、辅助交易决策
- **目标用户**: 期货交易员、分析师、风险管理者

### 1.2 功能特性
1. **实时性**: 分钟级数据更新，快速响应市场变化
2. **多维度**: 价格、交易量、相关指标综合监控
3. **可视化**: 直观展示趋势、相关性、因果关系
4. **可操作**: 提供明确的归因分析和决策建议

## 四、核心算法与计算策略

### 4.1 分层计算策略（性能优化核心）

#### 第一层：简单相关（实时计算）
- **计算内容**: 同期相关性（Pearson相关系数）
- **时间复杂度**: O(N×M²)，N=数据点，M=指标数
- **预期耗时**: 0.1-1ms（10指标×100数据点）
- **更新频率**: 每次数据刷新时计算

#### 第二层：带时滞相关（秒级计算）
- **计算内容**: 跨期相关性（时滞0-3期）
- **时间复杂度**: O(N×M²×L)，L=时滞数
- **预期耗时**: 2-20ms
- **更新频率**: 用户请求时计算

#### 第三层：格兰杰因果（后台计算）
- **计算内容**: 格兰杰因果检验
- **时间复杂度**: O(N²×M²×L) 或 O(N×M²×L²)
- **预期耗时**: 50-200ms
- **更新频率**: 定时计算（如每5分钟）或用户显式请求

### 4.2 增量更新算法
```python
class IncrementalCorrelation:
    def __init__(self):
        self.n = 0
        self.sum_x = 0
        self.sum_y = 0
        self.sum_xy = 0
        self.sum_x2 = 0
        self.sum_y2 = 0
    
    def update(self, x, y):
        # 增量更新统计量
        self.n += 1
        self.sum_x += x
        self.sum_y += y
        self.sum_xy += x * y
        self.sum_x2 += x * x
        self.sum_y2 += y * y
    
    def correlation(self):
        # 实时计算相关性
        if self.n < 2:
            return 0
        numerator = self.n * self.sum_xy - self.sum_x * self.sum_y
        denominator = math.sqrt(
            (self.n * self.sum_x2 - self.sum_x**2) *
            (self.n * self.sum_y2 - self.sum_y**2)
        )
        return numerator / denominator if denominator != 0 else 0
```

### 4.3 异常检测算法
```python
def detect_anomalies(data, window=20, threshold=2.0):
    """基于移动平均和标准差的异常检测"""
    anomalies = []
    for i in range(len(data)):
        if i < window:
            continue
        window_data = data[i-window:i]
        mean = np.mean(window_data)
        std = np.std(window_data)
        if std == 0:
            continue
        z_score = (data[i] - mean) / std
        if abs(z_score) > threshold:
            anomalies.append({
                'index': i,
                'value': data[i],
                'z_score': z_score,
                'timestamp': timestamps[i]
            })
    return anomalies
```

```javascript
// 热力图实现
function renderCorrelationHeatmap(matrix, indicators) {
    const container = document.getElementById('correlation-heatmap');
    const size = indicators.length;
    
    // 创建表格
    let html = '<table class="correlation-table">';
    
    // 表头
    html += '<tr><th></th>';
    indicators.forEach(ind => html += `<th>${ind}</th>`);
    html += '</tr>';
    
    // 数据行
    for (let i = 0; i < size; i++) {
        html += `<tr><th>${indicators[i]}</th>`;
        for (let j = 0; j < size; j++) {
            const value = matrix[i][j];
            const intensity = Math.abs(value); // 绝对值表示强度
            const color = value > 0 ? 'var(--teal)' : 'var(--coral)';
            const opacity = intensity;
            
            html += `<td style="background: rgba(${color}, ${opacity})">`;
            html += value.toFixed(2);
            html += '</td>';
        }
        html += '</tr>';
    }
    html += '</table>';
    
    container.innerHTML = html;
}
```

## 六、性能优化策略

### 6.1 计算优化
1. **增量计算**: 避免全量重算，只计算新增数据点
2. **分层策略**: 按需计算，实时层轻量，因果层后台
3. **缓存机制**: 热点结果缓存，减少重复计算
4. **并行计算**: 多指标相关性可并行计算

### 6.2 前端渲染优化
1. **Canvas渲染**: 图表使用Canvas而非SVG，提升性能
2. **虚拟滚动**: 大数据量时使用虚拟滚动
3. **懒加载**: 非首屏内容延迟加载
4. **分帧渲染**: 复杂图表分帧绘制，避免阻塞

### 6.3 内存管理
1. **数据分页**: 历史数据分页加载
2. **定时清理**: 过期数据自动清理
3. **对象池**: 复用DOM元素和图表实例
